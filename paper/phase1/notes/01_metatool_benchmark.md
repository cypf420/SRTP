# MetaTool Benchmark 阅读笔记

## 论文信息

- 标题：MetaTool Benchmark for Large Language Models: Deciding Whether to Use Tools and Which to Use
- 年份：2024
- 会议：ICLR 2024
- 链接：https://proceedings.iclr.cc/paper_files/paper/2024/hash/bc12914d66b41b6bfc2d3a5decdb498b-Abstract-Conference.html

## 它研究什么

这篇工作关注一个比“会不会调用参数”更前置的问题：**模型到底知不知道什么时候该用工具，以及该选哪个工具**。这和本课题的 wrong-tool 问题高度相关，因为它把工具使用意识和工具选择错误显式拆开了。

## 数据和任务

- 构建了 MetaTool benchmark
- 包含 ToolE 数据集
- 同时覆盖单工具和多工具场景
- 任务被拆成 tool usage awareness 和 tool selection
- 还专门设计了相似工具、特定场景、多工具组合等子任务

## 方法与指标

这篇论文本质上是 benchmark paper，不是因果解释 paper。它主要通过任务划分和误差统计来观察模型在哪些工具选择情境下失败。

## 最重要的结论

1. 许多模型在“是否该调用工具”与“该调用哪个工具”上仍明显不足。
2. 工具描述质量与工具列表长度会显著影响表现。
3. 即使是强模型，在相似工具和复杂选择场景中也会明显掉点。

## 对本项目的直接价值

这篇工作最值得借鉴的是它对**工具选择问题本身的明确问题化**。你的项目可以直接沿着这条线，把“工具使用意识不足”和“工具选择错误”都吸收到特征体系中，例如：

- 是否存在更窄但更合适的工具
- 工具列表长度
- 相似工具密度
- 描述重写前后表现变化

## 它没有解决什么

1. 没有研究参数幻构。
2. 没有研究 under-specified input 时是否应澄清。
3. 没有把错误转成因果图。
4. 没有从内部表示空间分析脆弱边界。

## 你可以怎么用

- 用它来支撑“工具选择不是已解决问题”。
- 用它来论证“工具描述和工具集合结构会影响错误率”。
- 把它当作你工具侧特征设计和描述增强防御的直接前置依据。
