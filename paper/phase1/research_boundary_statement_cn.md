# 研究边界说明

## 一、这项研究准备主张什么

本研究准备主张的是：

1. 在**单步 tool-calling**场景下，工具选择错误和参数幻构并非完全随机；
2. 在明确的数据构造、标签体系和特征假设下，可以得到一组**稳定且人工可读的关键驱动因子**；
3. 这些因子可以通过**因果结构学习 + 反事实干预**得到一定程度的机制解释；
4. 错误样本在开源模型的内部表示空间中，可能表现出**边界附近聚集或更高脆弱性**；
5. 基于关键因子和边界风险，可以构造一个**轻量前置防御层**，在一定程度上降低 wrong tool 和 parameter hallucination。

## 二、这项研究不准备主张什么

本研究不准备主张的是：

1. **不主张得到哲学意义上的最终因果真相。**
   这里的因果图是在给定数据、变量定义、先验约束和可观测条件下得到的结构性解释，不等于覆盖所有真实世界 agent 系统的绝对因果规律。

2. **不主张覆盖完整多步 agent 轨迹。**
   当前阶段只聚焦单步“用户请求 -> 工具选择 + 参数生成”，不试图一次性解决长链规划、记忆回写、环境反馈循环等完整 agent 问题。

3. **不主张对所有模型都做内部机制解释。**
   表征边界分析主要依赖能导出 hidden states 或 logits 的开源模型；闭源模型主要用于行为层面对比，而不是内部因果审计。

4. **不主张防御系统已经解决了 agent 安全。**
   这里的 defense wrapper 是 pre-call 风险控制原型，重点解决澄清、混淆工具增强和高风险样本识别，不等于完整安全框架。

5. **不主张实验数据天然代表真实世界全部分布。**
   数据中包含大量结构化构造样本，因此论文必须同时报告构造规则、偏差风险和外部验证结果，避免把“在合成高混淆场景上成立”误写成“在所有部署环境上成立”。

## 三、因此论文中的措辞应该怎么收

建议全文始终使用以下收敛表达：

- “在当前数据构造和特征假设下”
- “表明存在稳定关联与可干预因子”
- “支持一种结构性解释”
- “为前置防御提供依据”
- “不排除未观测混杂和分布迁移的影响”

不建议直接使用以下过强表述：

- “证明了最终因果机制”
- “解释了所有工具调用错误”
- “适用于任意 agent 系统”
- “彻底解决了 prompt injection / hallucination / tool misuse”

## 四、对后续实验设计的约束

这份边界说明会反过来约束实验：

1. 必须保留**train/dev/test/external_eval** 划分，避免一边调规则一边汇报结果。
2. 必须显式报告**误伤率、澄清轮数和外部验证集表现**，否则防御效果结论不完整。
3. 必须对因果图做**稳定性分析和反事实验证**，不能只展示单次跑图结果。
4. 必须把表示分析定位为**边界脆弱性证据**，而不是直接声称找到了“模型的真实思想”。

## 五、当前最合适的论文定位

最合适的定位不是“全能型 agent 安全论文”，而是：

> **一篇聚焦单步工具调用失败、强调可解释特征、因果结构学习、表示边界分析与轻量防御闭环的 workshop / short paper / early-stage systems paper。**

这个定位和你的时间、人力、可获得资源是匹配的，也更容易把结论讲得稳。
